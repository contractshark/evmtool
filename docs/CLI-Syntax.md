<h1><a id="user-content-hyperledger-besu-command-line" class="anchor" aria-hidden="true" href="#hyperledger-besu-command-line"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Hyperledger Besu command line</h1>
<p>This reference describes the syntax of the Hyperledger Besu Command Line Interface (CLI) options
and subcommands.</p>
<h2><a id="user-content-specifying-options" class="anchor" aria-hidden="true" href="#specifying-options"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Specifying options</h2>
<p>You can specify Besu options:</p>
<ul>
<li>On the command line</li>
<li>As an <a href="#besu-environment-variables">environment variable</a></li>
<li>In a <a href="/hyperledger/besu-docs/blob/1.5.4/docs/HowTo/Configure/Using-Configuration-File.md">configuration file</a>.</li>
</ul>
<p>If you specify an option in more than one place, the order of priority is command line, environment
variable, configuration file.</p>
<h3><a id="user-content-besu-environment-variables" class="anchor" aria-hidden="true" href="#besu-environment-variables"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Besu environment variables</h3>
<p>For each command line option, the equivalent environment variable is:</p>
<ul>
<li>Upper-case</li>
<li><code>_</code> replaces <code>-</code></li>
<li>Has a <code>BESU_</code> prefix</li>
</ul>
<p>For example, set <code>--miner-coinbase</code> using the <code>BESU_MINER_COINBASE</code> environment variable.</p>
<h2><a id="user-content-options" class="anchor" aria-hidden="true" href="#options"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Options</h2>
<p>To start a Besu node run:</p>
<div class="highlight highlight-source-shell"><pre>besu [OPTIONS] [COMMAND]</pre></div>
<h3><a id="user-content-auto-log-bloom-caching-enabled" class="anchor" aria-hidden="true" href="#auto-log-bloom-caching-enabled"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>auto-log-bloom-caching-enabled</h3>
<p>=== "Syntax"</p>
<pre><code>```bash
--auto-log-bloom-caching-enabled=false
```
</code></pre>
<p>=== "Environment Variable"</p>
<pre><code>```bash
BESU_AUTO_LOG_BLOOM_CACHING_ENABLED=false
```
</code></pre>
<p>=== "Example Configuration File"</p>
<pre><code>```bash
auto-log-bloom-caching-enabled=false
```
</code></pre>
<p>Enables or disables automatic log bloom caching. APIs such as
<a href="/hyperledger/besu-docs/blob/1.5.4/docs/Reference/API-Methods.md#eth_getlogs"><code>eth_getLogs</code></a> and
<a href="/hyperledger/besu-docs/blob/1.5.4/docs/Reference/API-Methods.md#eth_getfilterlogs"><code>eth_getFilterLogs</code></a> use the cache for improved performance.
The default is <code>true</code>.</p>
<p>If automatic log bloom caching is enabled and a log bloom query reaches the end of the cache, Besu
performs an uncached query for logs not yet written to the cache.</p>
<p>Automatic log bloom caching has a small impact on performance. If you are not querying logs blooms
for a large number of blocks, you might want to disable automatic log bloom caching.</p>
<h3><a id="user-content-banned-node-ids" class="anchor" aria-hidden="true" href="#banned-node-ids"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>banned-node-ids</h3>
<p>=== "Syntax"</p>
<pre><code>```bash
--banned-node-ids=&lt;bannedNodeId&gt;[,&lt;bannedNodeId&gt;...]...
```
</code></pre>
<p>=== "Command Line"</p>
<pre><code>```bash
--banned-nodeids=0xc35c3...d615f,0xf42c13...fc456
```
</code></pre>
<p>=== "Environment Variable"</p>
<pre><code>```bash
BESU_BANNED_NODEIDS=0xc35c3...d615f,0xf42c13...fc456
```
</code></pre>
<p>=== "Configuration File"</p>
<pre><code>```bash
banned-nodeids=["0xc35c3...d615f","0xf42c13...fc456"]
```
</code></pre>
<p>A list of node IDs with which this node will not peer. The node ID is the public key of the node.
You can specify the banned node IDs with or without the <code>0x</code> prefix.</p>
<p>!!!tip</p>
<pre><code>The singular `--banned-node-id` and plural `--banned-node-ids` are available and are two names
for the same option.
</code></pre>
<h3><a id="user-content-bootnodes" class="anchor" aria-hidden="true" href="#bootnodes"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>bootnodes</h3>
<p>=== "Syntax"</p>
<pre><code>```bash
--bootnodes[=&lt;enode://id@host:port&gt;[,&lt;enode://id@host:port&gt;...]...]
```
</code></pre>
<p>=== "Command Line"</p>
<pre><code>```bash
--bootnodes=enode://c35c3...d615f@1.2.3.4:30303,enode://f42c13...fc456@1.2.3.5:30303
```
</code></pre>
<p>=== "Environment Variable"</p>
<pre><code>```bash
BESU_BOOTNODES=enode://c35c3...d615f@1.2.3.4:30303,enode://f42c13...fc456@1.2.3.5:30303
```
</code></pre>
<p>=== "Example Configuration File"</p>
<pre><code>```bash
bootnodes=["enode://c35c3...d615f@1.2.3.4:30303","enode://f42c13...fc456@1.2.3.5:30303"]
```
</code></pre>
<p>A list of comma-separated <a href="/hyperledger/besu-docs/blob/1.5.4/docs/Concepts/Node-Keys.md#enode-url">enode URLs</a> for
<a href="/hyperledger/besu-docs/blob/1.5.4/docs/HowTo/Find-and-Connect/Bootnodes.md">P2P discovery bootstrap</a>.</p>
<p>When connecting to MainNet or public testnets, the default is a predefined list of enode URLs.</p>

<h3><a id="user-content-sync-mode" class="anchor" aria-hidden="true" href="#sync-mode"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>sync-mode</h3>
<p>=== "Syntax"</p>
<pre><code>```bash
--sync-mode=FAST
```
</code></pre>
<p>=== "Command Line"</p>
<pre><code>```bash
--sync-mode=FAST
```
</code></pre>
<p>=== "Environment Variable"</p>
<pre><code>```bash
BESU_SYNC_MODE=FAST
```
</code></pre>
<p>=== "Configuration File"</p>
<pre><code>```bash
sync-mode="FAST"
```
</code></pre>
<p>The synchronization mode. The options are <code>FAST</code> and <code>FULL</code>.</p>
<ul>
<li>The default is <code>FULL</code> when connecting to a private network by not using the <a href="#network"><code>--network</code></a>
option and specifying the <a href="#genesis-file"><code>--genesis-file</code></a> option.</li>
<li>The default is <code>FAST</code> when using the <a href="#network"><code>--network</code></a> option
with named networks, except for the <code>dev</code> development network. <code>FAST</code> is also the default if Ethereum mainnet
is being connected to by not specifying the <a href="#network"><code>--network</code></a> and <a href="#genesis-file"><code>--genesis-file</code></a>
options.</li>
</ul>
<p>!!! note</p>
<pre><code>When running Besu on some cloud providers, a known [RocksDB](https://github.com/facebook/rocksdb/issues/6435)
issue causes fast sync to fail occasionally. The following error is displayed repeatedly:

```
...
EthScheduler-Services-1 (importBlock) | ERROR | PipelineChainDownloader | Chain download failed. Restarting after short delay.
java.util.concurrent.CompletionException: org.hyperledger.besu.plugin.services.exception.StorageException: org.rocksdb.RocksDBException: block checksum mismatch:
....
```

The failure has been seen on AWS and Digital Ocean. A full restart of the AWS VM is required to
restart the fast sync. Fast sync is not [currently supported on Digital Ocean](https://github.com/hyperledger/besu/blob/master/CHANGELOG.md#143).
</code></pre>
<p>!!! important</p>
<pre><code>Using fast sync with
[private transactions](../../Concepts/Privacy/Privacy-Overview.md) or on Digital Ocean Droplets
is not supported.
</code></pre>
<h3><a id="user-content-target-gas-limit" class="anchor" aria-hidden="true" href="#target-gas-limit"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>target-gas-limit</h3>
<p>=== "Syntax"</p>
<pre><code>```bash
--target-gas-limit=&lt;INTEGER&gt;
```
</code></pre>
<p>=== "Command Line"</p>
<pre><code>```bash

```
</code></pre>
<p>=== "Environment Variable"</p>
<pre><code>```bash
BESU_TARGET_GAS_LIMIT=8000000
```
</code></pre>
<p>=== "Configuration File"</p>
<pre><code>```bash
target-gas-limit="8000000"
```
</code></pre>
<p>The gas limit toward which Besu will gradually move on an existing network, if enough miners are in
agreement. To change the block gas limit set in the genesis file without creating a new network,
use <code>target-gas-limit</code>. The gas limit between blocks can change only 1/1024th, so the target tells
the block creator how to set the gas limit in its block. If the values are the same or within
1/1024th, Besu sets the limit to the specified value. Otherwise, the limit moves as far as it can
within that constraint.</p>
<p>If a value for <code>target-gas-limit</code> is not specified, the block gas limit remains at the value
specified in the <a href="/hyperledger/besu-docs/blob/1.5.4/docs/Reference/Config-Items.md#genesis-block-parameters">genesis file</a>.</p>
